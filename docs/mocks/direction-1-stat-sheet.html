Rationale

The data is the interface. This direction styles the site like the reference stat platforms the audience already trusts (HLTV, Liquipedia): near-black cool surfaces, one workhorse font (Inter) with tabular numerals, and exactly two voices of color — yellow is reserved for player identity (Orbitron names, the Q in the wordmark, the points bar), blue for interactive controls. The medal system stays but grows up: header pills and low-alpha cell tints instead of paint-bucket columns, so 24 dense rows scan without noise. It is the safest direction to implement — almost everything is a literal MUI theme token — and its restraint is what makes the leaderboard feel authoritative rather than hobbyist. Filter chrome collapses into one toolbar; on phones rows become two-line cards and filters become a chip rail, so nothing is lost at 375px.

  • Rows become two-line cards. Line 1: rank, name, points (the three things that matter). Line 2: game logos, the four medal counts as color-coded pills — the medal column system survives as pills — plus event count. No horizontal scrolling, ever.
  • Filters become a chip rail. One horizontal row of chips (game, mode, years, LAN, POWER). Tapping a chip opens a bottom sheet with the full control (the year slider lives there at full width). The rail is sticky under the header.
  • Search docks into the header as an expanding icon field.
  • Tap a card → player page; the card layout reuses the exact desktop tokens, so it is one responsive render path in PlayerList, not a second design.

Implementation notes (MUI mapping)

  • Pure theme work: palette (bg/paper/primary #7aa7ff), shape.borderRadius: 8–12, typography = Inter + custom playerName style (Orbitron 800); font imports drop from 8 families to 2.
  • Medal classes (gold-header …) keep their names — values change to pill-style overrides; emotion-hash selectors die. Cell tints are the existing classes at new alphas.
  • LAN / POWER checkboxes restyled as MUI Chip/ToggleButton — presentational swap inside the existing FormControlLabel slots.
  • Points bar = ::after on the points cell (width % of leader) — one styleOverride, no new component.
  • Modes column renders “first two + +N” with the full list as tooltip — formatting only, sort logic untouched.
